üìã Oracle Agent Spec (YAML/JSON)

For: Production multi-agent systems

agent-spec/*.yaml

Use When:

  • Building production multi-agent systems that run 24/7
  • Need platform-agnostic deployment (AutoGen, LangGraph, WayFlow)
  • Coordinating multiple agents (orchestrator patterns)
  • Enterprise governance, auditing, security required
  • Long-running autonomous operations
Example: content-strategist.yaml 
agent:
  name: "content-strategist"
  version: "1.0.0"
  description: "AI agent for market trend analysis and content calendar management"

  metadata:
    industry: "creative-marketing"
    maturity_level: "bronze"
    autonomous_level: "high"

  config:
    model:
      provider: "anthropic"
      name: "claude-sonnet-4-5"
      temperature: 0.3

  tools:
    - name: "analyze_trending_topics"
      type: "mcp_server"
      mcp_server: "analytics-mcp"
      description: "Analyze trending topics from social media and search data"

    - name: "generate_content_calendar"
      type: "mcp_server"
      mcp_server: "content-management-mcp"

  mcp_servers:
    - name: "analytics-mcp"
      endpoint: "${MCP_ANALYTICS_ENDPOINT}"
      authentication:
        type: "oauth2"
        scopes: ["read:analytics"]

  workflows:
    - name: "daily_trend_analysis"
      schedule: "0 9 * * *"  # Daily at 9 AM
      steps:
        - analyze_trending_topics
        - generate_content_calendar

  governance:
    approval_required:
      - "publish_content"
    audit_all_actions: true

‚öôÔ∏è Workflow Automations (YAML)

For: Deterministic business processes

workflows/*.yaml

Use When:

  • Deterministic, rule-based processes (no AI decision-making)
  • Human approval gates required
  • Sequential/parallel task orchestration
  • Integration with business process automation tools
  • Clear if/then logic without AI reasoning
Example: campaign-launch.yaml 
workflow:
  name: "campaign-launch"
  version: "1.0.0"
  description: "Multi-channel marketing campaign launch automation"

  trigger:
    type: "manual"
    approver_role: "marketing_manager"

  steps:
    - id: "content_generation"
      agent: "content-strategist"
      action: "generate_campaign_content"
      parameters:
        campaign_id: "${workflow.input.campaign_id}"
      timeout: "5m"

    - id: "human_review"
      type: "human_approval"
      approver_role: "creative_director"
      notification_channels: ["email", "slack"]
      timeout: "24h"

    - id: "schedule_posts"
      condition: "approved"
      agent: "social-media-manager"
      action: "schedule_social_posts"
      parallel:
        - platform: "twitter"
        - platform: "linkedin"
        - platform: "facebook"

    - id: "launch_ads"
      agent: "campaign-orchestrator"
      action: "activate_ad_campaigns"
      requires: ["schedule_posts"]

    - id: "monitor_performance"
      agent: "analytics-agent"
      action: "track_campaign_metrics"
      duration: "7d"
      alerts:
        - metric: "ctr"
          threshold: "< 1.5%"
          action: "notify_manager"

Example: MCP Server Configuration 
# MCP Server: analytics-mcp

## Description
Google Analytics 4 and Adobe Analytics integration server

## Data Flow Pattern
Real-time streaming + hourly batch aggregation

## Endpoints
- `GET /metrics/realtime` - Real-time active users, page views
- `GET /metrics/campaigns` - Campaign performance data
- `POST /reports/custom` - Generate custom analytics reports

## Authentication
- Type: OAuth 2.0
- Scopes: `analytics.readonly`, `analytics.edit`
- Token refresh: Automatic

## Data Sources
- Google Analytics 4 API
- Adobe Analytics 2.0 API
- Custom event tracking via GA4 Measurement Protocol

## Usage Example (in Agent Spec)
```yaml
mcp_servers:
  - name: "analytics-mcp"
    endpoint: "${MCP_ANALYTICS_ENDPOINT}"
    authentication:
      type: "oauth2"
      client_id: "${ANALYTICS_CLIENT_ID}"
      client_secret: "${ANALYTICS_CLIENT_SECRET}"
      scopes: ["analytics.readonly"]

    retry_policy:
      max_attempts: 3
      backoff: "exponential"

    rate_limiting:
      requests_per_minute: 60
```

## Confluent Integration
Available as pre-built connector: `google-analytics-4-source`

üé≠ Multi-Agent Orchestration Patterns

Production multi-agent systems use sophisticated orchestration patterns where agents collaborate to achieve complex business outcomes. Each pattern serves specific use cases and team dynamics.

üîó Sequential Chain

Agents execute in sequence, passing results to the next agent. Best for workflows with clear dependencies (e.g., audit ‚Üí review ‚Üí deploy).

Agent A ‚Üí Agent B ‚Üí Agent C ‚Üí Result
Example: Smart Contract Deployment
1. Auditor analyzes code
2. Security Monitor validates
3. DeFi Analyst estimates gas
4. Deployment to testnet

‚ö° Parallel Processing

Multiple agents work simultaneously on different aspects of a problem, then results are merged. Optimizes for speed.

‚îå‚Üí Agent A ‚Üí‚îê
‚îÇ Agent B ‚îÇ‚Üí Merge ‚Üí Result
‚îî‚Üí Agent C ‚Üí‚îò
Example: Campaign Launch
‚Ä¢ Content Strategist: Creates posts
‚Ä¢ SEO Optimizer: Optimizes keywords
‚Ä¢ Social Media Manager: Schedules
‚Üí Campaign Orchestrator merges

üéØ Orchestrator-Worker

A central orchestrator agent coordinates and delegates tasks to specialized workers, then aggregates results. Best for complex multi-step processes.

üì° Event-Driven

Agents react to events asynchronously. Agents subscribe to event streams and trigger actions based on specific conditions.

Event Bus ‚Üí Agent subscribes ‚Üí Trigger action
Example: Security Monitoring

‚Üí Alert + Circuit breaker

üë§ How Humans Work with Agent Teams

üéØ

Strategic Direction

Humans set goals, priorities, and business constraints. Agents execute tactical operations autonomously within those guardrails.

Example: "Increase campaign ROI by 25%" ‚Üí Agents optimize channels, timing, content
‚úÖ

Approval Gates

Critical decisions require human approval (e.g., deploying to mainnet, spending >$10K). Workflows pause for human review.

Example: Agent audits contract ‚Üí Human reviews ‚Üí Agent deploys
üìä

Insight Consumption

Agents deliver insights via dashboards, Slack notifications, emails. Humans make strategic pivots based on agent analysis.

Example: Analytics Agent detects trend shift ‚Üí Alert ‚Üí Human adjusts strategy

üì¶ Production Deployment Steps

1

Define Agent Specifications

Create Oracle Agent Spec YAML files for each production agent. Define capabilities, MCP server connections, governance rules, and business KPIs. 

agent:
  name: "content-strategist"
  version: "1.0.0"
  description: "Market trend analysis and content calendar management"

  config:
    model:
      provider: "anthropic"
      name: "claude-sonnet-4-5"

  mcp_servers:
    - name: "analytics-mcp"
      endpoint: "${MCP_ANALYTICS_ENDPOINT}"
    - name: "social-media-mcp"
      endpoint: "${MCP_SOCIAL_MEDIA_ENDPOINT}"

  governance:
    approval_required: ["publish_content", "ad_spend>5000"]
2

Configure MCP Servers

Set up MCP servers for each data source. Configure authentication, rate limiting, retry policies, and health checks. Use Confluent pre-built connectors when available. 

mcp_server:
  name: "analytics-mcp"
  type: "google-analytics-4"

  authentication:
    type: "oauth2"
    client_id: "${GA4_CLIENT_ID}"
    client_secret: "${GA4_CLIENT_SECRET}"
    token_url: "https://oauth2.googleapis.com/token"

  rate_limiting:
    requests_per_minute: 60

  health_check:
    enabled: true
    interval_seconds: 30
3

Choose Orchestration Framework

Select runtime: AutoGen (research-focused), LangGraph (graph-based), WayFlow (enterprise), or custom. All can read Oracle Agent Spec.

AutoGen
Research, prototyping
LangGraph
Complex workflows
WayFlow
Enterprise governance
4

Deploy with Monitoring

Deploy to Kubernetes/ECS. Enable metrics (Prometheus), logs (Datadog), tracing (Jaeger). Set up alerts for failures and performance degradation. 

kubectl apply -f agent-deployment.yaml

# Monitor
kubectl logs -f deployment/content-strategist-agent
prometheus-query: agent_requests_total{agent="content-strategist"}
5

Test & Iterate

Run end-to-end tests. Monitor KPIs defined in Agent Spec. Iterate on prompts, orchestration logic, and MCP configurations based on real usage.

Best Practice: Start with Bronze-tier agents (high autonomy, clear KPIs), prove ROI in 3 months, then expand to Silver and Gold tiers.

MCP Integration Guide

Step 1: Install MCP Server

npm install @anthropic/mcp-server
# or
yarn add @anthropic/mcp-server

Step 2: Configure Environment Variables

export MCP_ANALYTICS_ENDPOINT="https://analytics.example.com"
export ANALYTICS_CLIENT_ID="your_client_id"
export ANALYTICS_CLIENT_SECRET="your_client_secret"

Step 3: Add to Agent Spec

mcp_servers:
  - name: "analytics-mcp"
    endpoint: "${MCP_ANALYTICS_ENDPOINT}"
    authentication:
      type: "oauth2"
      client_id: "${ANALYTICS_CLIENT_ID}"
      client_secret: "${ANALYTICS_CLIENT_SECRET}"
